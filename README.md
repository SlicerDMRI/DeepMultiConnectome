# DeepMultiConnectome

This repository contains the code used to train and evaluate [DeepMultiConnectome](https://doi.org/10.1016/j.neuroimage.2026.121765): Deep multi-task prediction of structural connectomes directly from diffusion MRI tractography. 

DeepMultiConnectome predicts the regions of interest a streamline connects to, which allows the generation of a structural connectome without the need for a gray matter parcellation. The model is able to predict labels for two atlases simultaneously, namely an 84 ROI Desikan-Killiany and a 164 ROI Destrieux based atlas, with the potential to extent the framework to more atlases. The predicted streamline node pair labels can be used to construct structural connectomes with different weightings, e.g. weighted by number of streamlines, mean FA, or SIFT2.

## Data preparation
This research used data from the publicly available HCP dataset, which can be accessed [here](https://www.humanconnectome.org/study/hcp-young-adult/). 
Tractograms and ground truth connectomes were generated by running `data/tractography.sh` on the HCP subjects. Running tractography requires [MRtrix3](https://mrtrix.readthedocs.io/en/latest/installation/package_install.html) and [FSL](https://fsl.fmrib.ox.ac.uk/fsl/docs/#). If you downloaded HCP zip files, you may want to restructure them first using `data/HCP_restructure_subjects.sh`.
To build training data, run `data/prepare_training_data.py`, which encodes streamline ROI labels and aggregates tractography outputs into train/val/test pickle files for fast loading. Set `output_dir` in that script to `TrainData_${input_data}` so it matches the `input_data` tag used by the train/test scripts.

## Training and testing
Training: `bash train_test/train.sh`

Testing: `bash train_test/test.sh`

`train.sh` uses the preformatted train/val/test pickle files from `data/prepare_training_data.py`, while `test.sh` runs on subject-level VTK tractography files and generates predicted connectomes during inference (analysis scripts are for downstream evaluation/plots).

The scripts in the analysis folder were used to generate the figures reported in the paper.
